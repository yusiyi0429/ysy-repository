# 测试数据使用指南

> 所有数据围绕「科技型企业普惠贷款」业务场景，按四步法管线步骤组织。

---

## 目录结构

```
data/samples/
├── README.md                    ← 你正在看的这份指南
├── step1-场景锚定/              ← Step1 输入数据
├── step2-文档萃取/              ← Step2 文档模式输入数据
├── step2-案例复盘/              ← Step2 案例复盘模式 + 模式发现 Skill 共用
├── step3-知识对齐/              ← Step3 专家修订输入数据
```

---

## Step1 — 场景锚定

**这个步骤做什么**：定义知识萃取的范围和结构，生成 Excel 骨架模板。

| 文件 | 怎么用 | 说明 |
|------|--------|------|
| `Step1_场景名称.txt` | 复制内容 → 粘贴到「场景名称」输入框 | "科技型企业普惠贷款全流程管理" |
| `Step1_场景内容.txt` | 复制内容 → 粘贴到「场景内容」输入框 | 业务范围的详细描述 |
| `Step1_子场景列表.csv` | 按 CSV 中的行，逐条在界面中添加子场景 | 3 个子场景：营销 / 贷后管理 / 不良分析 |
| `测试模板.xlsx` | 在「高级」区域上传，或选择「部门惯用表头」 | 传统 Excel 模板（可选） |

**操作路径**：填写场景名称和内容 → 添加子场景 → 保留或调整知识列 → 点击「生成场景骨架」

---

## Step2 — 知识萃取（3 种模式）

### 模式 A：📄 文档萃取

**这个模式做什么**：从制度文件/操作手册中自动提取结构化知识条目。

| 文件 | 怎么用 | 适用场景 |
|------|--------|---------|
| `科技企业贷后检查要点_轻量测试.txt`（~800字） | 上传或粘贴 → 选「标准萃取」 | 🔥 **首次测试用**，1 分钟内出结果，快速验证流程 |
| `科技企业普惠贷款业务操作指引.txt`（2.2万字） | 上传或粘贴 → 选「深度萃取」 | 深度测试，覆盖全流程，出 20-40 条知识 |

**操作路径**：选定「🔍 知识萃取」卡片 → 📄 文档模式 → 上传/粘贴文件 → 选萃取风格 → 执行

---

### 模式 B：📋 案例复盘

**这个模式做什么**：将专家踩坑经历以结构化表单输入，提取隐性信号。

| 文件 | 怎么用 | 信号类型 |
|------|--------|---------|
| `案例复盘_01_芯片企业流水异常.txt` | 逐个字段粘贴到案例复盘表单 | 直觉 + 反模式 |
| `案例复盘_02_生物医药破例审批.txt` | 同上 | 破例逻辑 |
| `案例复盘_03_软件企业法人变更.txt` | 同上 | 关系依赖 + 直觉 |
| `案例复盘_04_园区物业信息源.txt` | 同上 | 关系依赖 + 直觉 |
| `案例复盘_05_传统纺织汇率风险.txt` | 同上 | 跨领域对照（传统行业视角） |

**操作路径**：选定「🔍 知识萃取」卡片 → 切换到 📋 案例复盘 Tab → 按字段逐个粘贴 → 执行

---

### 模式 C：🔬 跨案例模式发现（独立 Skill）

**这个模式做什么**：对多个案例复盘交叉分析，发现反复出现的信号。

| 文件组合 | 怎么用 | 测试目标 |
|---------|--------|---------|
| 选 `案例复盘_01~04`（4 个科技企业案例） | 多文件上传（Ctrl+点击多选） | 发现科技企业领域内的共性风险信号 |
| 选 `案例复盘_01~05`（科技 + 传统） | 多文件上传 5 个 | 对比科技 vs 传统行业的信号差异 |
| 只选 `01 + 02`（2 个案例） | 最少案例数测试 | 验证最少 2 个案例的边界条件 |

**操作路径**：选定「🔬 模式发现」卡片 → 多文件上传或粘贴文本 → 执行 → 下载报告

---

### 模式 D：🎯 知识盲区检测（独立 Skill）

**这个模式做什么**：对比 Schema 列与实际填充率，发现知识空白区。

**不需要额外文件**，直接选定「🎯 盲区检测」卡片 → 执行即可。系统会自动读取当前流水线的 Step1 Schema 和 Step2 萃取结果。建议在用 `科技企业普惠贷款业务操作指引.txt` 完成文档萃取后执行——因为该文档侧重营销和贷后，不良分析子场景的列会自然形成盲区。

## Step3 — 知识对齐

**这个步骤做什么**：专家审核 Step2 萃取稿，逐条修订。

| 文件 | 怎么用 | 说明 |
|------|--------|------|
| `Step3_专家会议纪要.txt` | 复制内容 → 粘贴到「专家输入」文本框 | 3 位专家 4 条具体修订意见 |
| （留空） | 不填任何内容，直接点击执行 | 测试「无意见直通」功能 |

**操作路径**：确认 Step2 萃取稿已加载 → 粘贴专家意见（或留空） → 点击「发送并智能修订」 → 逐条审核 → 采纳时填写追问卡片 → 生成对齐稿

---

## Step5 — 智能转化 & 保鲜审计

不需要额外测试文件。完成 Step1-3 后，Step4/Step5 自动读取流水线中的知识 Excel。

| 功能 | 怎么用 |
|------|--------|
| 生成交付物 | Step5 点击「生成交付物」，获得 SKILL.md / QA 对 / 思维链 |
| 保鲜度审计 | Step5 点击「🔄 知识保鲜度审计」按钮 |

---

## 推荐测试路径

### 🚀 快速体验（8 分钟）

```
Step1: 名称 → 场景内容 → 3 个子场景 → 生成骨架
Step2: 选 📄 文档 → 上传「轻量测试.txt」 → 标准萃取 → 执行
Step3: 粘贴「专家会议纪要」 → 智能修订 → 采纳 2 条 → 生成对齐稿
Step5: 生成交付物
```

### 📊 深度验证（30 分钟）

```
Step1: 同上
Step2-a: 选 📄 文档 → 上传「操作指引.txt」 → 深度萃取 → 执行
Step2-b: 不离开页面 → 切 🎯 盲区检测 → 执行（观察不良分析盲区）
Step2-c: 切 📋 案例复盘 → 粘贴「案例复盘_01」 → 执行（观察隐性信号列）
Step2-d: 切 🔬 模式发现 → 多选 01~04 → 执行 → 下载报告
Step3:   进入 Step3 → 观察修订上下文侧栏 → 粘贴纪要 → 逐条修订+追问
Step5:   生成交付物 → 保鲜度审计
```

### 🔬 跨领域对照（额外 5 分钟）

```
Step2: 切 🔬 模式发现 → 多选 01~05（含传统纺织案例） → 执行
        对比报告中「科技企业特有信号」vs「通用信号」
```
