# 隐性知识显性化流水线 — 业务说明文档

**适用读者：** 业务负责人、领域专家、知识运营、项目组  
**文档性质：** 从业务目标、流程与价值解读本系统（非技术实现手册）  
**版本：** 20260525-optional-sub-scenario  
**更新日期：** 2026-05-26

---

## 1. 这套系统在解决什么问题？

### 1.1 背景

银行及大型机构里，大量关键能力藏在**专家经验**里：怎么审、怎么判、什么情况要 escalate、哪些坑不能踩。这些内容往往：

- 散落在制度 PDF、会议纪要、口头传授里；
- 新人上手慢，老人离职带走经验；
- 直接让大模型「自由发挥」容易胡编，难以审计、难以复用。

### 1.2 本系统的定位

**隐性知识显性化流水线**是一套「**把专家经验变成可交付知识资产**」的作业平台：

| 传统做法 | 本系统做法 |
|----------|------------|
| 专家从零填 Excel，负担大 | 先用制度/手册 **萃取一版初稿**，专家改而不是从零写 |
| Word/纪要改完再手工整理 | 专家意见 **结构化进 Excel**，保留修订痕迹 |
| 知识停在表格里 | 定稿后 **一键编译成 SKILL.md**，供智能体/助手按规则使用 |
| 每人一套格式 | **场景锚定模板**统一「这个领域知识长什么样」 |

一句话：**不是替代专家，而是把「写文档」变成「审稿 + 定稿 + 发布」的标准流水线。**

## 2. 谁在用？各自做什么？

| 角色 | 主要使用环节 | 业务职责 |
|------|--------------|----------|
| **知识运营 / 项目经理** | 全流程 | 建流水线、选模板、协调专家、把控进度与质量 |
| **领域专家** | Step3、Step4 | 对萃取稿提出修订；终审确认业务准确性 |
| **制度/合规提供方** | Step2 输入 | 提供制度、手册、案例等「已显性」材料 |
| **平台管理员** | 模型配置 | 接入内网大模型（如建行 ainlplm）或 OpenAI 兼容网关 |
| **下游消费方** | Step5 产出 | 使用 **SKILL.md** 配置智能助手、Agent、内部 Copilot |

## 3. 业务价值（为什么要五步法？）

五步法对应知识资产化的**五个质量关卡**，每一步都有明确的业务产出物：

```
制度/手册/纪要  →  场景骨架  →  萃取稿  →  修订稿  →  终审稿  →  SKILL 知识包
     (材料)         (结构)      (初稿)    (专家改)   (定稿)     (可发布)
```

| 价值 | 说明 |
|------|------|
| **结构先行** | Step1 先定「场景—子场景—知识列」，避免后面内容无法归类 |
| **降低专家成本** | Step2 把约 10% 已写在制度里的知识先填进表，专家面对「待审稿」 |
| **可追溯** | Step3/4 保留修订状态、原文与改后内容，便于审计与复盘 |
| **可发布** | Step5 用程序确定性生成 SKILL，避免定稿后再被模型「改花」 |
| **可并行多场景** | 每条「流水线」对应一个业务场景（如信贷审批、贷后管理），互不干扰 |

## 4. 五步法 — 业务解读

### Step 01 · 场景锚定

**业务问题：** 这个领域要萃取哪些知识？表头长什么样？

**您需要准备：**

- **场景名称**：如「科技型企业普惠贷款全流程管理」
- **场景内容**：业务边界、关键环节、知识特征（一段话说明「萃取范围」）
- **子场景**（可选）：如「客户筛选」「贷后监控」；不填也可，系统按单一场景组织
- **萃取模板**：系统默认模板，或上传本部门惯用的 Excel 模板

**系统做什么：**

- 把场景信息写入模板，生成 **「场景骨架 Excel」**（`template_*.xlsx`）
- 模板左侧固定 **锚定列**：场景、场景说明、子场景、子场景说明（后两项可选）
- 右侧为 **知识列**（环节、方法、判断逻辑等），由模板定义，后续步骤只往这些列填

**业务要点（界面原话）：** *「骨架错则全盘错——本步定义这个领域的知识应该长什么样。」*

**典型产出：** 一份带场景结构的空表 + 版式，供 Step2 填入知识。

---

### Step 02 · 知识萃取

**业务问题：** 制度、手册里已经写明的知识，如何快速变成表格式初稿？

**您需要准备：**

- Step1 的 **场景骨架**（自动关联）
- **知识来源**：制度 TXT/PDF/DOCX、操作手册、案例库，或直接粘贴正文
- **萃取风格**：标准 / 深度 / 精简（控制条数与字段完整度）
- 选用的大模型与 **知识萃取 Skill**

**系统做什么：**

- 阅读来源材料，按表头字段抽取结构化条目
- 写入 **「萃取稿 Excel」**（`preextract_*.xlsx`）
- 若模型有输出但无法落入表格（0 条），系统不会生成空文件误导您

**业务要点：** *「把已显性化的 10% 先固化——让专家面对的不是空白 Excel，而是一份待审稿。」*

**典型产出：** 8～40 条（视风格而定）带分类、描述、条件、逻辑等字段的初稿。

---

### Step 03 · 专家修订

**业务问题：** 专家在会上说的修改意见，如何落到表里且看得见改了什么？

**您需要准备：**

- Step2 的 **萃取稿**
- **专家修订意见**：会议纪要、口述要点、修订清单（文本或文件）
- **修订风格**：标准 / 严格 / 宽松（控制改动幅度与条数上限）

**系统做什么：**

- 对比「萃取稿 + 专家意见」，生成 **「修订稿 Excel」**（`revision_*.xlsx`）
- 表中可含修订状态、原始内容、修订内容等列，便于专家二次核对
- 支持 **在线编辑 Excel**（Luckysheet），人工微调后保存回流水线

**业务要点：** *「对比专家会议纪要与知识初稿，智能识别修订点并生成带标注的修订稿。」*

**典型产出：** 带修订痕迹的知识表，作为 Step4 输入。

---

### Step 04 · 最终确认

**业务问题：** 多轮专家意见后，如何形成「可归档、可对外发布」的定稿？

**您需要准备：**

- Step3 的 **修订稿**
- **其他专家指导内容**（可选）：交叉验证意见、补充纪要

**系统做什么：**

- 在修订稿基础上做 **终审整合**，生成 **「终审确认稿」**（`final_*.xlsx`）
- 可再次在线编辑，确保字段、分类、来源标注满足部门规范

**业务要点：** *「跨专家交叉验证 + 最终定稿归档。」*

**典型产出：** 部门认可的最终 Excel，仅此版本进入 Step5。

---

### Step 05 · 智能转化

**业务问题：** 定稿表格如何变成智能体能「按规矩办事」的知识包？

**您需要准备：**

- Step4 的 **终审确认稿**
- （可选）场景配置中的质量要求、是否输出反模式章节等

**系统做什么（重要：本步不再调用大模型「发挥」）：**

- 按确定性规则读取 Excel，编译为 **SKILL.md**
- 结构通常包括：场景概述、子场景一览（若有）、知识结构、反模式总结、使用示例等
- 可生成 **质量报告**（完整性、来源标注率等维度评分）

**业务要点：** *「Excel 是人工确认的真理源；SKILL 是发布给机器读的交付物。」*

**典型产出：**

- `SKILL_场景名.md` — 可导入 Agent / Cursor Skill / 内部知识库
- 质量报告 — 用于验收是否达到部门门槛（如条目数、反模式占比）

## 5. 核心概念（业务语言）

### 5.1 流水线

一条 **流水线** = 一个正在进行的 **场景萃取项目**。

- 可命名、保存、切换（如同时做「信贷审批」与「贷后管理」）
- 记录当前走到第几步、每步产出文件名
- 支持 **回退**：回到某步会清除该步之后的进度状态（磁盘上的历史文件需人工清理，见注意事项）

### 5.2 场景与子场景

| 概念 | 含义 | 是否必填 |
|------|------|----------|
| **场景** | 本次萃取的主题边界 | **必填** |
| **场景说明** | 业务背景、流程、决策类型 | 建议填 |
| **子场景** | 场景下的细分环节 | **可选**；不填则全文按「知识分类」组织 |
| **子场景说明** | 该细分环节的范围 | 随子场景 |

示例（来自系统样例）：场景为「科技型企业普惠贷款全流程管理」，可拆子场景「客户筛选」「贷后监控」等。

### 5.3 萃取模板（Excel 锚定）

模板是 **全流程的契约**：

- 左四列锚定「写在哪条场景/子场景下」
- 右侧列定义「每条知识要回答哪些业务问题」（环节、方法、适用条件、判断逻辑、反模式等）
- Step2 的模型输出 **列名必须与表头一致**，否则无法写入

**业务建议：** 模板由领域牵头人一次设计好，多项目复用，减少后期返工。

### 5.4 Skill 与模型配置

| 配置项 | 业务含义 |
|--------|----------|
| **模型配置** | 连接内网/外网大模型；建行环境填 Access_Key_Id、Tx-Code、Sec-Node-No 等 |
| **Skill 配置** | 萃取、修订、终审各阶段的「提示词能力包」，决定抽取风格与输出格式 |
| **Trace-Id / 流水号** | 由系统自动生成，用于网关排障，无需业务人员填写 |

## 6. 标准作业路径（推荐）

适合首次上手的 **端到端业务动线**：

1. **立项**：新建流水线，填写场景名称与场景内容  
2. **定版式**：选择或上传萃取模板 → 生成场景骨架 → 下载核对表头  
3. **备材料**：收集制度、手册、案例 → 进入 Step2 上传或粘贴  
4. **出初稿**：选「标准萃取」+ 萃取 Skill → 执行 → 下载萃取稿，运营初审条数与字段  
5. **专家会**：专家阅读萃取稿，整理会议纪要 → Step3 粘贴意见 → 执行修订  
6. **人工精修**：在线编辑修订稿，修正明显错误  
7. **终审**：Step4 汇入其他专家意见 → 生成终审稿 → 业务负责人签字认可（流程外）  
8. **发布**：Step5 编译 SKILL.md + 查看质量报告 → 交付智能体团队或知识库  

**耗时参考（因场景而异）：**

- Step1：0.5～1 小时（含模板对齐）  
- Step2：视文档长度与模型速度，十数分钟级  
- Step3～4：取决于专家意见复杂度，常需多轮  
- Step5：分钟级（程序编译）

## 7. 质量与风控（业务侧关注点）

系统通过「步骤隔离 + 文件名规则」降低混稿风险，业务上请关注：

| 检查点 | 说明 |
|--------|------|
| **一步一稿** | 萃取、修订、终审是不同文件，不要拿场景骨架当萃取稿改 |
| **Step1 变更** | 重新生成场景骨架会清空后续步骤状态，需重新跑 2～5 |
| **空稿** | 萃取 0 条不会生成空 Excel，应检查来源材料或 Skill/模型 |
| **来源标注** | 制度类知识建议保留「来源文档、原文摘录」，便于合规审计 |
| **反模式** | 配置可要求一定比例「踩坑提示」，避免知识库只有正面条款 |
| **终审门槛** | `scenario-schema.yaml` 中可配置最低条目数、置信度、Skill 评分线等目标值 |

详细缺陷与加固记录见：`docs/PRODUCTION_AUDIT.md`（偏实施与测试）。

完成一条流水线后，您应拿到：

| 阶段 | 交付物 | 用途 |
|------|--------|------|
| Step1 | 场景骨架 Excel | 归档版式、培训「表怎么填」 |

## 9. 典型应用场景

1. **信贷/风控制度萃取**  
   将审批指引、贷后管理办法变为可检索、可引用的 Skill，供审单助手使用。

2. **运营 SOP 显性化**  
   把分散的操作手册收成结构化知识，新人培训与质检抽检共用一版表。

3. **专家访谈后整理**  
   纪要进 Step3/4，与 Step2 制度萃取互补，形成「制度 + 经验」双源知识库。

4. **内网离线部署**  
   前端资源本地化，模型走内网网关（如 ainlplm），满足金融环境隔离要求。

## 11. 常见问题（业务视角）

**Q：没有子场景可以做吗？**  
可以。仅填场景名称与场景内容即可，Step5 会按「知识分类」组织正文。

**Q：专家不想看 Excel，只看 Word 怎么办？**  
流程仍以 Excel 为中枢；可从各步下载 xlsx 转 Word，或请专家在「在线编辑」中改表。

**Q：模型抽错了怎么办？**  
Step3 用专家意见覆盖；或在在线编辑器中直接改单元格，保存后进入下一步。

**Q：SKILL 生成后还能改吗？**  
应回到 Step4 改终审稿，再重新 Step5 编译，保证「Excel 为唯一真理源」。

**Q：多条流水线能同时进行吗？**  
可以，在总览页切换不同场景项目即可。

## 12. 一句话总结

> **本系统把「专家经验萃取」产品化为一条可重复的五步流水线：先锚定场景结构，再用制度萃取初稿，经专家修订与终审定稿，最后发布为可供智能体使用的 SKILL 知识包——让人做判断，让机器做整理与发布。**
